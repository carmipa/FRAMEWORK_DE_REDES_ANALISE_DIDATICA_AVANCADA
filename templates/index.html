🖥️ DICAS DE TERMINAL (Linux e Windows) Referência fixa

Use estes comandos para conferir no sistema operacional as mesmas informações exibidas no framework (IP, máscara, DNS, gateway, rotas e sub-redes).

🐧 Linux (Bash)
Comando Para que serve
ip a Mostra IPs e máscaras CIDR das interfaces.
nmcli device show Detalha DNS, gateway e parâmetros de rede.
ip route Exibe a tabela de rotas e gateway padrão.
ip neigh
arp -n		 Mostra tabela ARP/NDP (mapeamento IP ↔ MAC) para validar camada 2 e detectar anomalias.
traceroute 8.8.8.8 Exibe os saltos até o destino para análise de caminho e latência.
sudo apt install ipcalc
ipcalc 172.19.0.0/20		 Calcula rede, broadcast, wildcard e capacidade de hosts.
🪟 Windows (CMD/PowerShell)
Comando Para que serve
ipconfig Mostra IP e máscara decimal das interfaces.
ipconfig /all Mostra DNS, gateway, DHCP e MAC address.
route print -4 Exibe tabela de roteamento IPv4 para auditoria de rotas.
arp -a Lista cache ARP (IP ↔ MAC) para troubleshooting de gateway e suspeita de ARP spoofing.
tracert 8.8.8.8 Mostra rota por saltos até o destino para validar caminho na rede.
Get-NetIPAddress Consulta moderna em PowerShell para filtros e scripts.
Validação rápida de prova: no Windows use ipconfig /all + arp -a + tracert 8.8.8.8; no Linux use ip a + ip route + ip neigh + traceroute 8.8.8.8.
