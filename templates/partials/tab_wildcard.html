Wildcard na prática Cisco: é a máscara invertida (onde a máscara tem bit 1, a wildcard tem 0 — “precisa casar”). Em EIGRP e OSPF, o comando network usa endereço de rede + wildcard (não use máscara decimal aqui). Em ACLs, o segundo valor também é wildcard. Esta aba só aceita wildcards contíguas (equivalente a um único CIDR), como nos laboratórios típicos.
💡 Exemplos práticos (ACL / OSPF / EIGRP)
Use estes modelos para fixar o uso real da wildcard em comandos Cisco.
ACL (rede /24)
access-list 101 permit ip 192.168.1.0 0.0.0.255 any
OSPF (área 0)
router ospf 1
 network 10.0.0.0 0.0.0.255 area 0
EIGRP (AS 100)
router eigrp 100
 network 172.16.0.0 0.0.255.255
